# UNADESK

UNADESK is a local SPA for working with articles and text annotations. The application allows you to create, edit, view, and delete articles, highlight arbitrary text fragments, assign them a color, and save comments directly in the article text.

The project is implemented on Angular 21 with standalone bootstrap, stores data in `localStorage`, and does not use third-party UI libraries for dialogs, tooltips, or annotation mechanics.

## Возможности

- создание, редактирование, просмотр и удаление статей;
- отображение списка сохранённых статей с карточками и превью изображения;
- валидация прямой ссылки на изображение для статьи;
- выделение текста в статье с последующим созданием аннотации;
- выбор цвета подсветки и ввод комментария для выделенного фрагмента;
- просмотр комментария по hover/focus на подсвеченном тексте;
- редактирование и удаление существующих аннотаций;
- сохранение статей и аннотаций в `localStorage`;
- уведомления о результатах действий и базовая keyboard-accessibility.

## Как устроено приложение

### Основные пользовательские сценарии

1. Пользователь создаёт статью с заголовком, текстом и необязательной ссылкой на изображение.
2. Статья появляется в общем списке и сохраняется между перезагрузками страницы.
3. На экране просмотра пользователь выделяет текст, подтверждает создание аннотации и добавляет комментарий.
4. Подсветка и комментарий сохраняются и доступны для последующего редактирования или удаления.
5. При изменении текста статьи приложение предупреждает, что существующие аннотации будут удалены, так как они привязаны к символьным смещениям.

### Хранение данных

Приложение работает полностью локально и использует `localStorage`.

- статьи хранятся по ключу `text-annotator.articles`;
- аннотации хранятся по ключу `text-annotator.annotations`.

Аннотации привязаны к тексту статьи через абсолютные смещения:

- `startOffset`
- `endOffset`

Из-за этого при изменении текста статьи связанные аннотации могут стать невалидными и удаляются после подтверждения пользователя.

## Структура проекта

```text
src/app/
  entities/      доменные сущности статей и аннотаций
  features/      пользовательские сценарии управления статьями и аннотациями
  pages/         экраны списка, редактора и просмотра статьи
  shared/        общие утилиты, валидаторы, toast и DOM-сервисы
  widgets/       рендеринг контента статьи с подсветкой аннотаций
```

Ключевые экраны:

- `/articles` — список статей;
- `/articles/new` — создание статьи;
- `/articles/:articleId/edit` — редактирование статьи;
- `/articles/:articleId` — просмотр статьи и работа с аннотациями.

## Архитектурные заметки

- бизнес-логика вынесена в сервисы;
- состояние статей и аннотаций синхронизируется через Angular signals и RxJS;
- рендеринг статьи и вычисление подсвеченных сегментов отделены от UI;
- работа с выделением текста построена на стандартных браузерных API без сторонних библиотек.

## Тестирование

В проекте есть два уровня проверок:

- `npm test` — unit/component tests;
- `npm run e2e` — end-to-end сценарии на Playwright.
